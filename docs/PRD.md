# Product Requirements Document — 마주봄

## 1. 문제 정의

장애를 가진 사람과 보호자가 겪는 두 가지 문제:

1. **본인이 어떤 직종이 맞는지 모른다** → 특성 기반 AI 직종 매칭
2. **채용공고를 봐도 적합한 환경인지 판단할 수 없다** → 채용공고 적합도 자동 분석

---

## 2. 타겟 사용자

- **1차**: 장애인 보호자 (자녀의 취업을 돕는 부모)
- **2차**: 장애인 본인
- 차별점: 기존 서비스는 장애인 본인 대상 — 마주봄은 **보호자도 주 타겟**

---

## 3. 페이지 구조

| 라우트 | 화면 | 조건 |
|---|---|---|
| `/` | A. 검사 전 랜딩 | 비로그인 또는 검사 미완료 |
| `/` | B. 검사 후 대시보드 | 로그인 + 검사 완료 |
| `/profile` | C. 프로필 (뷰 전용) | 로그인 |
| `/survey` | D. 검사 작성 / 편집 | 로그인 |

---

## 4. 핵심 기능

### F-01: 화면 A — 랜딩

- 히어로: "우리 아이에게 맞는 일, 같이 찾아볼까요?"
- CTA: "무료로 검사 시작하기" → `/survey`
- 보조 문구: "약 3분 소요 · 회원가입 없이 체험 가능"
- 서비스 소개 3항목: 특성 검사 / AI 직종 매칭 / 공고 적합도 판별
- 신뢰 요소: 한국고용정보원 · 한국장애인고용공단 공공데이터 기반

### F-02: 화면 D — 검사 폼 (`/survey`)

**Step 1 — 기본 정보**

| 필드 | 입력 타입 | 선택지 | 필수 |
|---|---|---|---|
| 이름 | 텍스트 | — | ✅ |
| 성별 | 단일 선택 | 남성 / 여성 | ✅ |
| 최종학력 | 단일 선택 | 특수학교 초등부 / 특수학교 중등부 / 특수학교 고등부 졸업 / 일반학교 졸업 / 전공과 수료 / 기타 | ✅ |
| 희망 지역 | 단일 선택 | 시·도 17개 + 기타 | ✅ |

**Step 2 — 신체 조건 + 희망 활동**

| 필드 | 입력 타입 | 선택지 | 필수 |
|---|---|---|---|
| 장애 유형 | 복수 선택 (1개 이상) | 시각장애 / 청각장애 / 지체장애 / 언어장애 / 안면장애 / 지적장애 / 자폐성장애 / 기타 | ✅ |
| 기타 장애 내용 | 텍스트 | "기타" 선택 시만 노출 | 조건부 |
| 장애 등급 | 단일 선택 | 장애의 정도가 심함 / 장애의 정도가 심하지 않음 / 모르겠어요 | ✅ |
| 이동 | 단일 선택 | 자유로움 / 보조기구 사용 / 휠체어 사용 | ✅ |
| 손 사용 | 단일 선택 | 세밀한 작업 가능 / 큰 동작만 가능 / 어려움 | ✅ |
| 체력 | 단일 선택 | 4시간 이상 활동 가능 / 2~4시간 / 2시간 미만 | ✅ |
| 의사소통 — 말하기 | 단일 선택 | 일상 대화 가능 / 짧은 문장 가능 / 단어 수준 / 비언어적 소통 | ✅ |
| 의사소통 — 지시 이해 | 단일 선택 | 복잡한 지시 이해 / 2단계 지시 이해 / 단순 지시만 가능 / 시범 보여주면 가능 | ✅ |
| 희망 활동 | 복수 선택 (1개 이상) | 같은 일 반복하기 / 손으로 만들기 / 물건 정리·분류 / 몸 움직이기 / 컴퓨터·기기 다루기 / 동식물 돌보기 / 청소·세탁 등 환경 관리 / 기타 | ✅ |
| 기타 활동 내용 | 텍스트 | "기타" 선택 시만 노출 | 조건부 |

**모드 전환**

| 모드 | 폼 초기 상태 | 제출 버튼 |
|---|---|---|
| 작성 (`/survey`) | 모든 필드 비어있음 | "검사 완료" |
| 편집 (`/survey?mode=edit`) | `profiles`에서 기존 데이터 로드 | "수정 완료" |

**제출 동작**
- `profiles` 신규 생성 또는 업데이트
- AI 직종 매칭 결과 생성 → `match_results` 저장
- 제출 후 `/` 대시보드로 이동

**이탈 방지**
- 입력값이 있는 상태에서 페이지 이탈 시도 → 확인 모달

### F-03: 화면 B — 대시보드 (`/`)

**검사 결과 요약**
- 레이더 차트 (5축): 반복 작업 선호 / 대인관계 / 신체 활동 / 세밀한 손작업 / 환경 민감도
- AI 한 줄 요약
- 매칭 직종 TOP 3: 적합도 라벨 + 직종명 + 매칭률(%)

**필터**

| 필터 | 선택지 |
|---|---|
| 적합도 | 전체 / 잘 맞아요 / 도전해볼 수 있어요 / 힘들 수 있어요 |
| 지역 | 전체 / 유저 희망 지역 기준 시군구 목록 |

**맞춤 채용공고 목록**

카드 1개 표시 항목: 적합도 배지, 고용 형태, 공고 제목, 회사명, 지역, 경력, 마감일, 급여, 북마크 토글

카드 클릭 → 공고 원문 외부 링크

### F-04: 화면 C — 프로필 (`/profile`)

- 좌측 사이드 메뉴 (모바일: 탭)

**사이드 메뉴**

| 메뉴 | 동작 |
|---|---|
| 검사 결과 (기본 활성) | 검사 결과 탭 표시 |
| 스크랩 | 북마크한 공고 목록 탭 표시 |

**탭 1 — 검사 결과**
- 기본 정보 (읽기 전용): 이름, 성별, 학력, 희망 지역
- 신체 조건 (읽기 전용): 장애 유형, 장애 등급, 이동, 손 사용, 체력, 말하기, 지시 이해
- 희망 활동 (읽기 전용): 선택된 항목을 태그로 나열
- AI 성향 분석: 레이더 차트 + 요약 텍스트 + 매칭 직종 TOP 3
- 검사 내용 수정 버튼 → `/survey?mode=edit`

**탭 2 — 스크랩한 공고**
- 북마크된 공고 목록 (대시보드 카드와 동일 형태)
- 북마크 클릭 → 즉시 해제
- 빈 상태: "아직 스크랩한 공고가 없어요" + "맞춤 공고 보러가기" → `/`

### F-05: AI 직종 매칭

- 입력값 기반 프로필 벡터화 → pgvector 유사도 매칭
- 적합도 라벨: "잘 맞아요" / "도전해볼 수 있어요" / "힘들 수 있어요"
- 레이더 차트 5축 수치(0–100) + 요약 텍스트 + TOP3 직종 생성
- 채용공고별 적합도 자동 분석
