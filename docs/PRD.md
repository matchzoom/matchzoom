# Product Requirements Document — 마주봄

## 1. 문제 정의

장애를 가진 사람과 보호자가 겪는 두 가지 문제:

1. **본인이 어떤 직종이 맞는지 모른다** → 특성 기반 AI 직종 매칭
2. **채용공고를 봐도 적합한 환경인지 판단할 수 없다** → 채용공고 적합도 자동 분석

---

## 2. 타겟 사용자

- **1차**: 장애인 보호자 (자녀의 취업을 돕는 부모)
- **2차**: 장애인 본인
- 차별점: 기존 서비스는 장애인 본인 대상 — 마주봄은 **보호자도 주 타겟**

---

## 3. 페이지 구조

| 라우트 | 화면 | 조건 |
|---|---|---|
| `/` | A. 검사 전 랜딩 | 비로그인 또는 검사 미완료 |
| `/` | B. 검사 후 대시보드 | 로그인 + 검사 완료 |
| `/profile` | C. 프로필 (뷰 전용) | 로그인 |
| `/survey` | D. 검사 작성 / 편집 | 로그인 |

---

## 4. 핵심 기능

### F-01: 화면 A — 랜딩

- 히어로: "우리 아이에게 맞는 일, 같이 찾아볼까요?"
- CTA: "무료로 검사 시작하기" → `/survey`
- 보조 문구: "약 3분 소요 · 회원가입 없이 체험 가능"
- 서비스 소개 3항목: 특성 검사 / AI 직종 매칭 / 공고 적합도 판별
- 신뢰 요소: 한국고용정보원 · 한국장애인고용공단 공공데이터 기반

### F-02: 화면 D — 검사 폼 (`/survey`)

**Step 1 — 기본 정보**
- 이름, 만 나이, 성별, 최종학력
- 희망 지역 1순위 (시·도 → 구·군 2단 연동), 2순위 (선택)
- 이동 경로 베리어 프리 (true/false)

**Step 2 — 신체 조건**
- 장애 유형: 지적장애 / 자폐성장애 / 기타 발달장애
- 장애 등급: 경도 / 중등도 / 중도 / 모르겠어요
- 이동, 손 사용, 체력, 의사소통(말하기), 의사소통(지시 이해)

**Step 3 — 희망 활동 유형**
- 복수 선택 (최소 1개): 같은 일 반복하기 / 손으로 만들기 / 물건 정리·분류 / 몸 움직이기 / 컴퓨터·기기 다루기 / 동식물 돌보기 / 청소·세탁 등 환경 관리 / 기타

**모드 전환**
- 작성 모드: 빈 폼, "검사 완료" 버튼
- 편집 모드 (`?mode=edit`): 기존 데이터 로드, "수정 완료" 버튼

**제출 동작**
- `child_profiles` 신규 생성 또는 업데이트
- 입력값 벡터화 → pgvector 저장
- AI 직종 매칭 결과 생성

### F-03: 화면 B — 대시보드 (`/`)

- 성향 요약: 레이더 차트 (5축) + AI 한 줄 요약
- 매칭 직종 TOP 3: 적합도 라벨 + 직종명 + 매칭률
- 필터: 적합도 / 근무 형태 / 지역
- 맞춤 채용공고 목록 (무한스크롤)
  - 카드: 회사 로고, 북마크, 적합도 라벨, 근무 형태, 지역, 공고 제목, 급여, 마감일, 매칭 포인트 요약

### F-04: 화면 C — 프로필 (`/profile`)

- 좌측 사이드 메뉴 (모바일: 탭)
- 탭 1 "내 검사 결과": 기본 정보 + 신체 조건 + 희망 활동 + AI 분석 결과
- 탭 2 "스크랩한 공고": 북마크 목록, undo 토스트
- 메뉴: 내 검사 결과 / 스크랩한 공고 / 검사 내용 수정 / 회원 탈퇴

### F-05: AI 직종 매칭

- pgvector 유사도 기반 직종 매칭
- 적합도 라벨: "잘 맞아요" / "도전해볼 수 있어요" / "힘들 수 있어요"
- AI 채용공고 적합도 분석

## 6. 외부 API (확정 아님)

| API | 용도 | 호출 시점 |
|---|---|---|
| 워크넷 직업정보 API | 직업별 수행 환경 데이터 | 주기적 배치 → DB 적재 |
| 직무데이터사전 API | 직종별 필요 지식·기술·태도 | 주기적 배치 → DB 적재 |
| 장애인고용공단 직업훈련 정보 | 발달장애인 직종별 취업률 | 주기적 배치 → DB 적재 |
| AI API | 성향 분석, 공고 환경 추출, 매칭 설명 생성 | 검사 완료 시, 공고 분석 시 |
